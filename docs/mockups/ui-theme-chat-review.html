Conversation Modes (#73)

System prompt varies by mode. Visual indicator in toolbar.

Mode Behavior

💬 Chat
Default conversational mode. No special system prompt modification. Standard chat behavior.
📋 Plan
Structured output. System prompt appends: "Respond with structured plans, checklists, and actionable steps."
🤖 Agent
Proactive tool use. System prompt appends: "Use tools proactively to research and verify."
Current scope: prompt-only branching (Level A).
Post Provider v2 upgrade path:
- Level B: mode controls allowedTools in query() — chat=no tools, plan=outline tools, agent=full vault tools
- Level C: each mode runs a distinct agent pipeline with different maxTurns, mcpServers, and systemPrompt

Building Level C now on the old Vercel AI SDK would mean creating new pipelines that Provider v2 will replace entirely.

Input History Navigation (#81)

Ctrl+Arrow Up/Down to navigate through previously sent messages.

Interaction Flow

1. User types "How to link notes?" and sends
2. User types "What is PARA method?" and sends
3. User types "Compare Zettelkasten vs PARA" and sends
4. User presses Ctrl+↑ → textarea fills with "Compare Zettelkasten vs PARA"
5. User presses Ctrl+↑ again → "What is PARA method?"
6. User presses Ctrl+↓ → back to "Compare Zettelkasten vs PARA"
7. User presses Ctrl+↓ again → restores original draft (empty or whatever was typed)

Implementation Details

State: inputHistory: string[] (max 50), historyIndex: number (-1 = not navigating), draftInput: string (stashed current text)

Session-scoped: History clears on page reload. Not persisted to vault.

Exit: Any typing (not arrow) resets historyIndex to -1, exiting history navigation mode.
