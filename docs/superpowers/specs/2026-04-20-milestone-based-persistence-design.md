# Milestone-Based Analysis Persistence

## Problem

Saved AI analysis documents contain only brief Process Log and Plan Outline, missing full report sections, executive summary, graph data, and follow-up questions. Root causes:

1. **Early-save fires when data is empty** — `v2PlanSections.length > 0` triggers save, but all sections are `pending` with no content; `buildV2AnalysisSnapshot` filters `status === 'done'` → empty report
2. **Incremental persist gated on React lifecycle** — `useEffect` in `useAIAnalysisResult.ts:240` requires `analysisCompleted` AND React component mounted; modal close → component unmount → persist effect destroyed
3. **BackgroundSessionManager doesn't persist** — `markCompleted()` updates status + sends Notice, never writes vault file
4. **`markCompleted()` fires before report generation** — agent stream ending ≠ report generated. Sections are generated by `ReportOrchestrator` AFTER user approves the plan.

## Key Insight: Two-Phase Architecture

```
Phase 1: Agent Stream → plan appears → stream ends → markCompleted()
         (sections are all 'pending', content is empty)

Phase 2: User approves plan → handleApprovePlan() → ReportOrchestrator.generateReport()
         → all sections stream in parallel → summary agent runs
         (NOW sections have content, summary exists)
```

Save must happen after Phase 2, not Phase 1.

## Design

### Principle

**Discrete milestone saves at content completion, not at agent stream events.**

### Save Points

| Milestone | Trigger | Content Saved |
|-----------|---------|---------------|
| **Plan ready** | `vault_submit_plan` processed | Process Log (done steps) + Plan Outline + Sources |
| **Agent done** (background only) | `BGM.markCompleted()` | Updated Process Log (all steps done) — overwrites early save |
| **Report generated** | `handleApprovePlan()` → `generateReport()` returns | Everything: Process Log + Plan + Report Sections + Summary + Graph |
| **Section regenerated** | `handleRegenerateSection()` returns | Overwrite with updated section content |

### Architecture

Service-layer function `persistSessionToVault(snapshot, options)` — no React deps:

```
persistSessionToVault(snapshot, options) → Promise<string | null>
  ├── buildV2AnalysisSnapshotFromData(snapshot)
  ├── buildCompletedSnapshotFromV2()
  ├── fromCompletedAnalysisSnapshot() → buildMarkdown()
  └── saveToVault / persistToPath
```
