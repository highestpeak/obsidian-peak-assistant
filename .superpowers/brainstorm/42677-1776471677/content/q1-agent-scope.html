AI Agent 应该生成什么层次的数据？

决定 Topology / Bridges / Timeline 三个 tab 的数据生成架构

A

一次 Agent 生成全部图数据 (推荐)

一次 agent 调用输出完整的结构化 JSON：节点角色、边类型+语义标签、主题聚类、时间演化链。前端按 tab 过滤和布局。

Pros

  • 延迟最低 — 切换 tab 无需等待
  • 数据一致性 — 三个视角共享同一份推理结果
  • Token 消耗最少 — 源文件内容只发送一次

Cons

  • 单次调用输出较大的 JSON
  • 如果用户只看某一个 tab，其余推理"浪费"了
B

每个 Tab 独立 Agent 调用

Topology Agent 推断关系结构，Bridge Agent 做主题聚类+桥梁识别，Timeline Agent 做思想演化链。各 tab 独立调用、独立流式展示。

Pros

  • 按需调用 — 只在用户切换时触发
  • 每个 agent prompt 更聚焦
  • 可以并行调用

Cons

  • 3x token 消耗（源文件重复发送）
  • 数据可能不一致（不同 agent 对同一文件的理解不同）
  • 切换 tab 有等待延迟
C

分层：基础 Agent + 按需二次推理

一次基础调用生成通用图数据（节点关系、分组），切换到 Bridge tab 时再补一轮轻量推理（识别桥梁节点），切换到 Timeline 时再推理演化链。

Pros

  • 基础数据快速可用
  • 深度分析按需触发

Cons

  • 架构复杂 — 需要管理基础+增量两层状态
  • 二次推理仍需等待
  • 增量推理可能与基础数据矛盾
