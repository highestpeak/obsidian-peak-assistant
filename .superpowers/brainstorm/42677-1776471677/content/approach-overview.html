AI Graph Agent 设计方案

一次 Anthropic Agent SDK tool-use loop，生成三个 tab 的全部图数据

Agent 架构

数据流
用户搜索 → Top Sources (13 files) → 点击 Graph 按钮 → Graph Agent Loop → 结构化 JSON → 三个 Tab
Graph Agent (Anthropic SDK)
System Prompt: 搜索查询 + source 文件列表 + 基础 metadata
任务: 分析文档关系，输出图结构
可用工具 (3个):
read_document
读取文件全文内容
get_linked_documents
获取双向 wikilink 列表
submit_graph
提交最终图结构 JSON
典型 loop: 读 3-5 个关键文件 → 查 links → 推理关系 → submit_graph (3-6 轮)

Agent 输出 Schema (submit_graph 参数)

GraphOutput JSON
{ nodes: [{ path, label, role: "hub" | "bridge" | "leaf", cluster: "AI产品", // AI推断的主题聚类 summary: "一句话摘要", importance: 0.9 // 映射节点大小 }], edges: [{ source, target, kind: "builds_on" | "contrasts" | "complements" | "applies" | "references", label: "B 将 A 的理论落地为产品方案", weight: 0.8 }], clusters: [{ id: "ai-product", name: "AI产品设计", description: "关于AI产品策略和飞轮设计的文档" }], bridges: [{ path: "AI-IdeaStream-how-to-know...", connects: ["AI产品设计", "个人成长"], explanation: "将创业验证方法论与个人项目管理结合" }], evolution_chains: [{ chain: [path1, path2, path3], // 按思想演化排序 theme: "AI产品想法从构思到验证的演化" }] }
