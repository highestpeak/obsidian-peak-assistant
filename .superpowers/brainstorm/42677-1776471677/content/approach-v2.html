AI Graph Agent 设计方案 v2

搜索阶段即生成图，批量读取，Anthropic Agent SDK

搜索 Pipeline 集成

时序流程
用户搜索 → AI Search → Top Sources → Graph Agent (并行/串行)
↑ 搜索结果一出来就启动 graph agent
↑ 不等用户点 Graph 按钮
结果：用户切到 Graph 视图时，数据已经准备好
• 如果 agent 还在跑 → 显示 loading + 进度
• 如果已完成 → 直接渲染图
• 图数据缓存在搜索结果中，切换 List/Graph 不重新调用

Agent 设计

Graph Agent (Anthropic SDK tool-use loop)
初始 Context:
• 搜索查询 (用户的问题)
• source 文件列表 (path + 文件名 + 文件夹 + ctime/mtime)
工具 (2个):
read_sources
参数: paths: string[]
返回: {path, content, outgoing_links[], incoming_links[]}[]
一次读取所有源文件内容 + 双向 links
submit_graph
参数: GraphOutput JSON
作用: 提交最终图结构，结束 agent loop
前端收到后渲染三个 tab
典型 Agent Loop (2轮):
Turn 1: Agent 调用 read_sources(所有路径) → 获得全部内容和 links
Turn 2: Agent 分析关系、聚类、桥梁、演化链 → 调用 submit_graph(JSON)
总计: 2次工具调用，1次 LLM 深度推理

submit_graph 输出 Schema

GraphOutput — 一份数据供三个 Tab
{ nodes: [{ path: "AI-IdeaStream-产品飞轮...", label: "AI产品飞轮与自动化设计", role: "hub" | "bridge" | "leaf", cluster_id: "ai-product", summary: "探讨AI产品的飞轮效应和自动化策略", importance: 0.9 // → 节点大小 }], edges: [{ source: "path-A", target: "path-B", kind: "builds_on" | "contrasts" | "complements" | "applies" | "references", label: "B 将 A 的理论落地为产品方案", weight: 0.8 // → 边粗细 }], clusters: [{ id: "ai-product", name: "AI产品设计", description: "关于AI产品策略和飞轮设计的文档" }], bridges: [{ node_path: "AI-IdeaStream-how-to-know...", connects: ["ai-product", "personal-growth"], explanation: "将创业验证方法论与个人项目管理结合" }], evolution_chains: [{ chain: ["path-early", "path-mid", "path-latest"], theme: "AI产品想法从构思到验证的演化" }] }

三个 Tab 如何消费这份数据

Topology 关系图
数据: nodes + edges + clusters
节点大小: importance
节点颜色: cluster_id
边样式: kind → 颜色/虚实
布局: dagre, hub 居中
Tooltip: "展示文档间的语义关系和知识结构"
Bridges 知识桥梁
数据: clusters + bridges
布局: 泳道式，cluster 分组
桥梁节点: 跨泳道连线 + 解释
非桥梁: 收在所属 cluster 内
Tooltip: "标识跨越知识领域的关键连接文档"
Timeline 演化时间线
数据: evolution_chains + ctime
布局: 水平时间轴 + 日期刻度
演化链: 按 chain 连线
间距: 真实时间比例
Tooltip: "展示知识积累和思想演化的时间脉络"
